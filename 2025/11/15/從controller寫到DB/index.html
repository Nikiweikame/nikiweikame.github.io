從controller寫到DB

從controller寫到DB的五個步驟

感覺這部分需要從MVC開始說起

view是前端的畫面
model是屬於資料處理的部分，從商業邏輯到寫入DB
controller則是串接view與model的橋樑

view-controller-model
而controller-model又再往下細分
Controller → Service → Repository → Model → DB

Controller：流程、協調者

不放商業邏輯
不處理資料庫
只負責「流程」與「接收／回傳」

Service：商業邏輯的核心

處理流程中的複雜邏輯
例如：
計算運動熱量
比對使用者權限
交易流程（建立訂單、扣庫存、寄通知）
Controller 只叫 Service，以保持乾淨

Repository 的用途是：

專門處理 DB CRUD
讓 Service 不直接碰 ORM
將來換 DB（MySQL → MongoDB）也不用改 Service

Model（ORM）：資料模型

負責操作 DB
包含關聯（hasMany、belongsTo…）
不能塞太多邏輯，避免膨脹

DB（資料庫）

最後的儲存層
